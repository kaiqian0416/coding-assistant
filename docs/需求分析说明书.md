## 1. 引言

### 1.1 项目名称

**AI驱动的编程练习助手**（AI-Powered Coding Practice Assistant）

### 1.2 项目背景

随着人工智能技术的快速发展，AI在教育领域中的应用日益广泛。传统编程学习平台主要提供固定题库和标准答案，难以针对学生个体差异提供个性化指导。

本项目旨在开发一个基于Web的AI驱动编程练习助手，通过智能分析用户的编程能力、错误类型及学习进度，为用户提供个性化练习推荐、代码错误诊断和学习路径规划服务，提高编程学习效率。

### 1.3 项目目标

开发一个Web应用，实现以下核心能力：

1. **用户管理**：注册、登录、密码修改、个人信息维护
2. **在线编程练习**：Python代码在线编写、提交、自动判题
3. **AI代码诊断**：接入智谱AI大模型，分析代码错误并给出修复建议
4. **AI代码审查**：防止"面向结果编程"（如直接print(8)应付两数之和）
5. **个性化推荐**：根据知识点掌握度自动推荐薄弱环节题目
6. **学习数据统计**：多维度展示学习进度、正确率、知识点掌握度
7. **管理员后台**：题库管理、AI辅助出题、用户管理、成就管理
8. **特色功能**：排行榜、打卡签到、成就系统、错题本

### 1.4 预期成果

- 可运行的Web系统，包含注册登录、编程练习、AI诊断、推荐、统计功能
- 完整的数据库设计与SQL脚本
- 前后端分离架构，RESTful API接口
- 深色/浅色双主题UI
- 完整的技术文档

## 3. 功能需求分析

### 3.1 用户管理模块

#### 3.1.1 注册
- **前置条件**：未登录状态
- **输入**：用户名（2-50字符）、密码（≥6字符）、昵称（可选）、邮箱（可选）
- **处理**：BCrypt加密密码，检查用户名唯一性，空邮箱转null
- **输出**：注册成功/失败提示

#### 3.1.2 登录
- **输入**：用户名、密码
- **处理**：BCrypt验证，成功存入Session
- **输出**：管理员→后台首页；普通用户→学员首页
- **特效**：Canvas粒子背景+毛玻璃登录框+登录成功烟花+欢迎弹窗（区分管理员/学员）

### 3.2 编程练习模块

#### 3.2.1 题库浏览
- 按难度（简单/中等/困难）筛选
- 按知识点（基础语法/条件判断/循环结构等10个分类）筛选
- 以等宽等高卡片网格展示，每个卡片显示标题、难度标签、知识点标签、描述摘要

#### 3.2.2 代码编辑与提交
- **编辑器**：深色背景代码编辑区，预设Python模板
- **提交**：POST请求，携带question_id、code、language('python')
- **判题流程**：
  1. 创建提交记录（status='judging'）
  2. 真实Python执行：将代码写入临时.py文件，调用系统python3/python执行，传入样例输入，捕获stdout
  3. 若Python不可用或执行异常，回退到模拟判题模式（分析print()语句）
  4. 与预期输出比对：完全匹配=100分，部分匹配=30-50分
  5. AI代码审查：判断是否为真实解题vs硬编码，硬编码最多15分
  6. 若有错误，AI诊断分析+建议
  7. 更新学习统计（知识点维度的正确/错误计数+正确率）

### 3.3 AI辅助功能

#### 3.3.1 AI代码诊断
- 调用智谱AI（GLM-4-Plus）API
- 提示词：用3-5句话简要分析，直接指出问题在哪行、怎么改
- 未配置API Key时自动使用模拟模式（内置规则分析）

#### 3.3.2 AI代码审查（防面向结果编程）
- 检测纯硬编码（如print(8)应付两数之和）
- 内置规则：无input()调用且无变量定义且含print() → 硬编码
- API模式：发送给AI判断，返回JSON `{"isGenuineSolution": true/false, "feedback": "一句话评价"}`
- 硬编码→平台分数降到最高15分

#### 3.3.3 AI辅助出题（管理员）
- 输入知识点、难度、主题
- AI返回JSON格式：题目描述、样例输入、样例输出、参考答案
- 自动解析（自动清理```json标记），存入数据库，状态='pending'
- 管理员审核后发布

### 3.4 学习辅助模块

#### 3.4.1 个性化推荐算法
1. 从learning_stats获取用户各知识点掌握度
2. 找出掌握度最低的3个知识点
3. 按用户ability_level匹配难度
4. 从question表随机抽取对应题目（5道）
5. 无历史数据时默认推荐"简单"难度基础语法题

#### 3.4.2 学习统计
- 概览卡片：总做题数、通过数、错误数、正确率
- 知识点掌握度：每个知识点的正确率进度条
- 薄弱知识点：按正确率升序排列TOP3
- 提交历史：最近20条，可展开查看代码、错误、AI诊断

#### 3.4.3 排行榜
- 按通过题数降序排名（排除管理员）
- 前三名展示🥇🥈🥉奖牌
- 当前用户行高亮
- 公开访问（无需登录）

#### 3.4.4 打卡签到
- 今日有提交→✅已打卡，显示今日提交次数
- 首页卡片展示

#### 3.4.5 成就系统
- 数据库驱动的可配置成就
- 条件类型：通过数≥N、错误数≥N、总提交数≥N
- 进度条展示，已获得→✅+绿色

#### 3.4.6 错题本
- 自动收集所有未通过提交
- 展开查看代码、错误信息、AI诊断
- 一键复制代码

### 3.5 管理员模块

| 功能 | 描述 |
|------|------|
| 题目管理 | 表格展示、按难度/状态筛选、新增/编辑/删除题目 |
| AI出题 | 输入要求→AI生成→预览→审核通过/驳回 |
| 审核 | 待审核题目列表、通过/驳回操作 |
| 提交记录 | 查看所有用户提交，按用户名/结果筛选，展开查看详情 |
| 用户管理 | 搜索、新增、编辑（角色/密码/信息）、删除（不可删除自己） |
| 成就管理 | 增删改成就要素（名称/图标/条件类型/阈值） |

## 4. 非功能需求

### 4.1 性能要求
- 页面加载时间 ≤ 3秒
- 代码提交判题响应 ≤ 15秒（含AI诊断）
- 支持50并发用户

### 4.2 安全性
- 用户密码BCrypt加密存储（成本因子10）
- Session登录状态管理
- 管理员权限校验（Controller层+拦截器层双重校验）
- 空邮箱转null避免UNIQUE约束冲突
- 管理员禁止提交代码（前后端双重拦截）

### 4.3 可用性
- 界面简洁友好，操作流程清晰
- 深色/浅色双主题切换
- 注册→做题→看诊断全程 ≤ 2分钟

### 4.4 可靠性
- 系统运行稳定，连续24小时无崩溃
- 异常处理：全局异常处理器统一返回中文错误信息

### 4.5 可扩展性
- 模块化分层架构，后续可增加编程语言支持
- 成就系统数据库驱动，管理员可自由配置新成就
- AI服务接口化，可切换不同大模型

### 4.6 可维护性
- 统一代码规范：Lombok简化POJO，MyBatis-Plus统一数据访问
- 清晰的分层结构：Controller→Service→Mapper→Entity
- API文档化：RESTful路径命名规范

- 预置15道Python编程题，一键启动即可使用
