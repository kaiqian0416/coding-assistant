## 幻灯片 5：判题引擎 —— 核心技术亮点

## ⚙️ 判题引擎设计

```
提交代码 → ① Python真实执行 → ② 输出比对
             ↓ 失败则回退
             模拟判题(分析print语句)
           ↓
         ③ AI代码审查（防硬编码）
             ├── 真实API: 智谱AI判断
             └── 模拟: 检测input()/变量/print()
           ↓
         ④ 诊断+反馈+更新学习统计
```

**亮点：**
- 🔄 **双模式**：真实执行优先，Python不可用时自动回退
- 🛡️ **防作弊**：`print(8)` 直接拦截，最高15分
- 📊 **实时统计**：每次提交自动更新知识点掌握度

## 幻灯片 6：AI集成 —— 三大AI场景

## 🤖 AI能力矩阵

| 场景 | 调用时机 | 提示词优化 | API Key？ |
|------|---------|-----------|-----------|
| **错误诊断** | 代码错误时 | "3-5句话，直接指出哪里错怎么改" | 无→模拟 |

**技术细节：**
- 统一`AiService`接口，`AiServiceImpl`单类管理
- JSON自动清理markdown标记（` ```json ``` `）
- `max_tokens=1200` 平衡回复长度
- 智谱AI/DeepSeek可切换（OpenAI兼容格式）

## 幻灯片 7：数据库设计

## 🗄 数据库设计（5张核心表）

```
user ─1:N─→ submission ←─1:N─ question
  │                               
  └─1:N─→ learning_stats
              
achievement (独立配置表)
```

| 表 | 关键字段 | 亮点 |
|----|---------|------|
| user | password(BCrypt), role, ability_level | 空邮箱→null避免UNIQUE冲突 |
| question | review_status(pending→approved) | 审核流程保证质量 |
| submission | result, score, ai_diagnosis | 完整记录提交全生命周期 |
| learning_stats | accuracy(动态计算) | 每次提交自动UPDATE |
| achievement | condition_type + condition_value | **数据库驱动、管理员可配置** |

## 幻灯片 10：代码质量 —— 规避三大缺陷

## 🛠 规避往届三大开发缺陷

### ❌ 缺陷1：代码书写风格混乱

| 问题 | 我们的解决方案 |
|------|--------------|
| 命名不一致 | 统一驼峰/下划线命名，MyBatis-Plus自动映射 |
| POJO冗余 | Lombok @Data简化getter/setter |
| 硬编码颜色 | CSS变量20+主题色统一管理 |
| 魔法值 | 枚举常量化（user/admin, approved/pending等）|

### ❌ 缺陷2：模块耦合严重

| 问题 | 我们的解决方案 |
|------|--------------|
| 前后端强耦合 | RESTful API + 统一ApiResponse |
| Service直接依赖 | 面向接口编程（5个接口+5个Impl）|
| AI切换困难 | AiService接口抽象 |

### ❌ 缺陷3：测试覆盖不足

| 问题 | 我们的解决方案 |
|------|--------------|
| 无测试 | 64个测试用例，100%通过 |
| 异常无处理 | GlobalExceptionHandler全局异常处理 |
| 参数无校验 | JSR-303注解校验 |
